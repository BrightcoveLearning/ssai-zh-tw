網址創建

網址模板

Brightcove Once UX利用基於Once VOD技術的專有URL語法，允許URL創建在功能上與標準的One URL非常相似。在一次UX中創建URL的唯一區別是更新的主機名以及在典型的一次DeliveryType和RequestedFileType變量之前包含兩個附加變量，如下所述。有關創建URL的其他說明，請參閱一次入門：一旦URL創建。

提供內容變量

主持人 AdServiceFlag AdServiceType
Onceux.unicornmedia.com 廣告 vmap

主持人 /服務/ 廣告服務標誌 / 廣告服務類型 /DeliveryType/RequestedFileType/DomainGUID/ApplicationGUID/MediaItemGUID/VirtualFileName.Extension

VMAP廣告匯總

這投放上面的內容變量對於“一次UX"，請使用服務的“廣告聚合"方法。此方法從通過匯總或 聚合 基於您配置的 applicationguid 和廣告服務器的廣告查看數據。收到“UX用戶體驗"響應後，此方法將提供元數據和一個VMAP對象，其中包含整個資產查看會話的廣告負載，該對像已標準化為VAST 3.0，然後嵌入到VMAP 1.0 XML對像中。通過此VMAP響應，您可以處理與用戶體驗（廣告素材或廣告盒倒數，隱藏播放器控件），信標和其他廣告素材類型（橫幅廣告，點擊廣告和VPAID廣告）相關的客戶端廣告功能。此VMAP對像還包含Brightcove Once XML對象，該對象包括Brightcove Once數據和用於播放的廣告插入的Ones URL。見VMAP樣本響應示例想要查詢更多的信息。

動態廣告參數

一旦UX允許使用所有允許的動態廣告參數（DAP）來填充廣告和信標調用，就像使用一次平台一樣。有關這些值的說明，請參見一次入門：動態廣告參數。

查詢字符串參數

一旦UX使用與一次平台相同的查詢字符串參數，本節將介紹三個參數，這些參數可用於輔助對一次UX響應和行為進行額外的微調。請參閱一次入門：查詢字符串參數有關其他可用查詢字符串參數的更多信息。

禁用跟踪事件

UMTP = {integer}

統一傳輸協議用於覆蓋或禁用默認的 Brightcove 服務器端廣告視頻展示和事件跟踪行為。

  • UMTP = 0

默認情況下，Brightcove代表客戶端/應用程序向廣告服務器觸發線性廣告視頻展示和跟踪事件。如果明確設置了參數，則“一次"將不會觸發展示或跟踪事件。

輸出

輸出= {integer}

此鍵值對控制Brightcove返回的響應的對象格式。默認情況下，一旦UX將使用XML進行響應，因此如果需要JSON響應，則應使用output = 1。

  • XML = 0
  • JSON = 1

定時文本擴展

定時文本擴展= {extension}

用於修改timedtext自適應/ M3U8和“一旦UX <uo：timedTextURL>"響應的輸出字幕類型。一旦UX在VMAP響應中以DFXP格式的xml文件為默認情況下無法使用Applevvtt的播放器/應用/環境提供了默認的sidecar。請參閱一次UX XML對象示例，以獲取此文件在VMAP響應中的位置。

可用的擴展名是：

  • srt
  • vtt（WebVTT）
  • xml（使用.xml擴展名的DFXP）
  • dfxp

uo：unicornOnce節點說明

節點 描述
特爾 插入此特定請求中的廣告的生存時間（以秒為單位）。如果在此時間之外提出請求，則不會插入任何廣告。最佳做法建議是，如果用戶等待ttl開始播放，則請求頂級一次UX URL。
內容 一次URL視頻響應將被回放到最終用戶。包含遵循標準“一次"行為的請求/確定格式的動態插入的廣告。
內容長度 未貨幣化基礎資產的持續時間（以秒為單位）。
有效載荷長度 獲利基礎資產的持續時間（以秒為單位）。筆記：VPAID持續時間不包括在此值中。
上下文ID 特定返回的內容和廣告有效負載的唯一值。
綠洲（內容查詢字符串） 附加到contenturi URL上的查詢字符串，其中包含contextid值，以指示該請求的特定內容和廣告有效負載。

uo：contentImpressions節點說明

節點 描述

時間 觸發contentImpression URL的時間（以秒為單位）。（將一直是0因為這是在媒體開始播放時觸發）。

uo：requestParameters節點說明

鑰匙 致電廣告提供商的鍵值對中的鍵

節點 描述
虛擬地圖：廣告來源 標識廣告插播屬性並充當海量數據
VMAP: 儲存庫資料 表示VMAP響應中將緊隨VAST 3.0廣告響應
vmap：AdBreak屬性說明

虛擬地圖：廣告中斷是一個重複的 VMAP 元素，其中包含您的廣告提供商收到的特定於每個廣告的信息。對於內容（pre0，mid0，mid1，post0）內的廣告中斷位置和中斷/ pod（ad0，ad1，ad2）內的廣告序列，AdBreaks將始終按時間順序進行排序。

節點 描述
中斷ID 廣告的VMAP名稱。該值指示休息時間類型，基於時間偏移的休息時間組標題以及休息時間的從零開始的時間位置值
時間偏移 從基本視頻內容的開始到廣告暫停在基本視頻內容中的位置之間的時間偏移。滾動前和滾動後的休息時間將分別顯示開始或結束的時間偏移。對於視頻插播廣告，時間偏移值必須與該特定中斷之前顯示的所有廣告的累積持續時間相結合，以便準確計算中斷。對於後貼片廣告插播，插播開始時間為內容長度結合之前展示的所有廣告創意的累計持續時間。

vmap：AdSource屬性說明

節點 描述
識別碼 整個VMAP文檔中廣告計數的從零開始的枚舉值

vmap：VASTData字段說明

媒體文件 VPAID資產資源的直通位置。筆記：視頻創意媒體文件將不會出現在VMAP對像中，因為“一次性XML"對像中提供的“一次性URL"會將這些素材資源插入正確的位置。
