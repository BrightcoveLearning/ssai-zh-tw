    影片雲端 SAI 概觀

    在本主題中，您將學習如何將 Brightcove 伺服器端廣告插入 (SSAI) 用於收錄動態傳遞的影片。

    概覽

    能夠播放視訊的裝置數量每天都會增加。OTT 連接設備, 特別是, 繼續以快速的速度取代傳統的電視.在這種環境中，為每個設備平台和每個廣告服務器或網絡構建客戶端播放器的整合可能是昂貴和耗時的。有時候為某些平台構建客戶端玩家並不實際。

    Brightcove SSAI 提供一種簡單的方式，將廣告支援的視訊傳遞給這些裝置，而不必為每個裝置建置和維護昂貴的整合。Brightcove SSAI 將廣告直接拼接到視訊內容串流中，因此它可以在任何可播放串流視訊的裝置上運作，以及每個工作階段廣告鎖定、用戶端指標、可視性報告，以及與熱門廣告服務的相容性 — 全都在領先業界的 Video Cloud 線上影片線上影片內容平台。

    注意事項：

    • 若要使用 SSAI，您的視訊雲端帳戶必須設定為動態傳遞並啟用 SSAI。請連絡您的客戶經理，開始使用此功能。
    • SSAI 可與隨選視訊 (VOD) 搭配使用，並支援即時播放。
    • 有關限制的列表，請參見下一節。

    處理流程

    如果沒有 SSAI，如果瀏覽器中有廣告攔截器，則使用 VAST、VMAP 和 VPAY 的 IMA 或飛輪等客戶端廣告將不會播放。

    啟用 SSAI 和自動容錯移轉並啟用廣告封鎖偵測功能後，您的廣告一律會播放：

    • 當沒有廣告攔截器時，將播放客戶端廣告。

    • 偵測到廣告攔截器時，就會播放伺服器端廣告。

    如果您未啟用 SSAI 的自動容錯移轉功能，則只會播放伺服器端廣告。

    為什麼是 SSAI？

    SSAI 的好處包括：

    • 流暢的電視般的用戶體驗 -獲得更高的參與度，而不會出現客戶端廣告的微調，錯誤和凍結。它只是播放-就像在普通電視上一樣！
    • 多螢幕觸及率 -是否想通過廣告支持的VOD來訪問OTT連接的設備？將您的影片推送到成長最快的視訊觀看區段，並輕鬆獲利。SSAI 適用於任何可以播放視頻的設備，布萊特灣播放器可選！
    • 恢復阻止廣告的損失 -立即恢復廣告屏蔽者損失的收入的10-30％。SSAI 無法像客戶端廣告一樣被阻止。
    • 每會話定位 -在每次廣告調用時傳遞視頻元數據標籤，設備信息，用戶ID或特定於應用程序的參數，以使個性化廣告定位能夠為您的廣告資源獲得最高的每千次展示費用。

    與布萊特灣播放器的 SSAI 提供了這些額外功能（僅 VOD）：

    • 客戶端信標 -從客戶端報告印象，參與度和可見度，包括與第三方分析庫的集成。
    • 禁用廣告搜索 -禁用廣告間隔中的快進。
    • 廣告點擊率 -廣告期間，用戶可以點擊以了解更多信息。
    • 隨播廣告疊加層 -線性視頻廣告可以包含伴隨重疊圖像。

    用戶端平台

    Brightcove SSAI 提供了一個簡單的方式，將廣告支持的視頻傳遞到 OTT 設備。

    用於Web和Dynamic Delivery API的Brightcove Player也可以用於許多自定義OTT集成，但是不支持這些集成除非通過與Brightcove全球服務（BGS）的諮詢安排。

    如果可以在排除裝置特定整合邏輯的簡單範例中展示問題，將會解決 API 互動或 Web 播放器功能問題。

    基於工作階段的目標

    廣告可以針對不同的平台、網站、應用程式，或根據播放時已知的其他標準進行評估，在個別播放工作階段發生時進行評估。您可以使用不同的廣告代碼來分割不同的廣告播放環境，也可以使用單一廣告代碼與廣告巨集。

    巨集會在廣告請求時進行評估，因此每次播放影片時，都可以參照從用戶端傳遞的變數，或根據視訊中繼資料或 HTTP 要求標頭資訊由 SSAI 服務插入。有關宏如何與Brightcove SSAI一起使用的詳細信息，請參見SSAI配置指南。

    SSAI 是避免網路上廣告封鎖程式的好方法。如果您想將SSAI與Brightcove Player一起使用，但又想在不被阻止的情況下繼續使用客戶端技術（例如VPAID廣告素材），請嘗試啟用我們的自動adblock檢測和故障轉移功能。

    可視性

    轉碼

    廣告變得可檢視之前，首先需要進行轉碼。然後，它可以被縫合到流中。這也適用於基於會話的目標。

    例如，如果您有兩個廣告 A 和 B，用於一個廣告配置：

    • 使用者 1 獲得廣告 A：開始轉碼；廣告不適用於使用者 1
    • 用戶 2 獲得廣告 A（幾分鐘後）：由於廣告 A 的轉碼已完成，因此他們會看到廣告
    • 使用者 1 取得廣告 B：開始轉碼；廣告不適用於使用者 1
    • 使用者 3 取得廣告 A：由於廣告 A 的轉碼已完成，因此他們會看到廣告
    • 使用者 4 取得廣告 B：由於廣告 B 的轉碼已完成，因此他們看到廣告

    廣告與內容無關。它們依賴於轉譯。因此，在上述範例中，如果使用者 2 觀看的內容與使用者 1 觀看的內容相同，則會收到廣告 A。這表示在上傳新內容時，通常不需要重新轉換任何廣告。

    第三方測量結果

    第三方度量和可視性系統可使用用戶端或伺服器端信標，與 Brightcove SSAI 搭配使用。適用於網路和 SDK 播放程式的 Brightcove 播放程式會發出用戶端事件，可供用戶端外掛程式直接報告。數家供應商已更新可見性和指標外掛程式以接收這些資料-請聯絡您的客戶經理以取得詳細資訊。

    先決條件

    SSAI 需要布萊特灣動態交付。這意味著您需要執行以下操作：

    1. 請連絡您的客戶經理，以啟用您的帳戶進行動態交付和 SSAI。
    2. 使用動態傳送內嵌設定檔，擷取一或多部新影片或重新轉碼舊影片。有關詳細信息，請參見概述：Dynamic Ingest API動態交付文件。

    限制

    使用 SSAI 外掛程式時，請注意下列限制：

    • SSAI 外掛程式僅與布萊特灣播放器 v6.8.0 及更新版本相容。v5 不提供任何支援。
    • SSAI要求您的內容視頻具有音頻，但不支持多個音軌。
    • 我們的系統第一次看到廣告時，直到廣告轉碼並準備好發送後才會播放。
    • 使用 SSAI 播放共用的視訊時，SSAI 巨集取代將使用父視訊中的中繼資料，而不是子視訊。如果上層影片標示為，即使子視訊標示為Advertising='Free'，SSAI 也會略過廣告查閱Ad Supported。

    入門

    若要開始使用 SSAI 與內嵌動態傳遞的視訊，請依照下列步驟執行：

    1. 向您的廣告供應商或廣告行動團隊取得廣告標籤。請確定與此代碼相關聯的廣告需求設定為不傳回 VPAD 廣告單元。

    2. 使用網頁瀏覽器，將廣告標籤貼到網址列，以要求廣告代碼。如果有替換參數，您可以嘗試使用假值的請求或保留變量名進行測試。

      • 如果收到錯誤回應，請先檢閱廣告代碼並確認廣告代碼正確無誤。您可能必須嘗試宏/變量的假值。
      • 看看響應類型。確認這是一個 DFP 廣告規則回應，顯示所有廣告分段，連結至每個網繭內廣告的 VAST 文件。回應應該說VAST要么VMAP在XML的前幾行中。他們可能會使用DFP中的其他XML 廣告規則XML。
      • 如果您不是使用 Free輪，您可以將這些值複製並貼到瀏覽器的網址列中，以手動方式操作重新導向。尋找MediaFiles指向兼容視頻源的元素。您正在尋找 MP4 或 HLS (.m3u8) 視訊來源檔案。
      • 如果廣告響應指向以結尾的文件.js，則它是VPAID廣告單元，不適用於Brightcove SSAI。
    3. 決定廣告代碼中的哪些參數可以由 SSAI 伺服器取代，以及哪些參數需要從播放器或裝置傳遞。適當地撰寫廣告標籤。有關廣告代碼宏和適當語法的詳細信息，請參見SSAI廣告配置文件。

    4. 決定是否要從客戶播放器或伺服器為您的廣告發出曝光和參與度信標。

      • 如果您的播放程式只能在無法使用 Brightcove 播放器 (網頁或 SDK) 的裝置上使用，或者您的目標是啟用廣告封鎖的網路案例，則您想要使用伺服器端信標。
      • 如果要在某些平台上從服務器發出信標，而在其他平台上從客戶端發出信標，則需要創建多個SSAI廣告配置。
         

      SSAI 廣告組態可以啟用或停用伺服器端信標。如果您使用的是布萊特灣播放器（網絡或 SDK），則還必須在客戶端上進行設置。

      確保不要發出兩次信標。
    5. 使用您的廣告標籤和信標設定建立 SSAI 廣告組態 (廣告設定)。如需詳細資訊，請參閱設定伺服器端廣告設定文件。
    6. 無論您的目標是哪個平台，您都可以先嘗試網路播放器做為測試。有關詳細信息，請參見使用Brightcove Player實施服務器端廣告文件。
    7. 確認播放器正常工作後，您可以根據以下內容諮詢適當的文檔，以定位其他平台：上面的平台表。
